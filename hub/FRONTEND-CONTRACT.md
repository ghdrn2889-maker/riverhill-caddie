# 프론트엔드 계약 — 디자인 AI ⇄ 구현 AI

> **목적:** 디자인 AI가 화면(`public/index.html`+CSS)을 소유하고, 구현 AI(Claude Code)가 데이터만 배선한다.
> 이 계약대로 만들면 구현 AI가 **CSS를 한 글자도 안 고치고** 그대로 배포한다 → 드리프트 없음.
> ← [프로젝트 허브](PROJECT.md) · [데이터 계약](CONTRACTS.md)

## 분업 원칙

- **디자인 AI가 만드는 것:** 실제 배포되는 `public/index.html`(마크업+CSS) 또는 **모든 상태를 채운 정적 레퍼런스**. 목업이 아니라 진짜 파일 형식.
- **구현 AI가 하는 것:** 그 화면의 지정된 **요소 ID**에 `/api/*` 데이터를 꽂는다(`app.js`). CSS·구조는 안 바꾼다.
- 바꿔야 할 이유(데이터가 없는 상태 등)가 생기면 → **사용자에게 말하고 결정**. 말없이 재해석 금지.

## 제약 (반드시)

- 자체완결: 외부 CDN·웹폰트 URL 금지(CSP 유사). 한글=시스템 폰트 스택.
- PWA·안드로이드 크롬·**폰 세로 전용**(본문 최대폭 ~600px). 360px에서 가로 스크롤 금지.
- 접근성: 터치 타깃 ≥44px, 색만으로 상태 구분 금지, 시각 숫자 tabular-nums, `prefers-reduced-motion` 존중.
- 기존 기능 보존: 알림 구독·소식 피드·근무기록(사진·CSV·PDF).

## 반드시 다뤄야 하는 화면 상태 (프로토타입이 빠뜨렸던 것 포함)

오늘 화면(상황판 히어로 + 행동 보드)은 아래 **모든** 경우를 각각 또렷하게:
1. **근무 확정 + 티오프 있음** — 집출발/도착/티오프 시각 + 카운트다운 + 진행 레일. (`/api/today` commute 있음)
2. **근무 확정 + 티오프 아직 없음** — "근무 확정 · 티오프 배정 대기"(시간 지어내지 않음)
3. **스페어 대기** — 순번 표시, "확정되면 알려드림". ← 매일 보는 상태. **프로토타입이 안 그렸던 부분, 꼭 디자인 필요.**
4. **지금 차례(your_turn)** — 최고 강조
5. **휴무(off)** — 조용한 톤
6. **미상/정보 없음(empty)** — 안내
7. **로딩·오류** — 이전 정상값 유지, "확인 못했어요 · 다시 시도"(오류를 휴무로 표현 금지)

## 구현 AI가 데이터를 꽂는 요소 ID (현재 index.html 기준)

디자인 AI가 새로 만들 때 **이 ID들을 유지**하면 구현 AI가 그대로 배선한다(다르게 하려면 이 표를 갱신):

| 영역 | 요소 ID | 들어가는 데이터 |
|---|---|---|
| 헤더 날짜·시각 | `#date` | "M월 D일 요일 · HH:MM" |
| 히어로 제목 | `#heroTitle` | 상태 라벨(오늘 근무 확정/스페어 등) |
| 히어로 부제 | `#heroSub` | 상태 설명 문장 |
| 행동 보드 컨테이너 | `#boardSlot` | 상태별 보드 HTML(근무=카운트다운, 그 외=안내) |
| 감시 상태 | `#hWatch` | "● 일정 감시 정상 · N분 전" |
| 알림 전달 | `#hPush` | "● 이 폰 알림 정상/꺼짐" |
| 알림 켜기 버튼 | `#enableBtn` (+`#enableMsg`) | 미구독 시 노출 |
| 오늘 소식 미리보기 | `#todayNews` | 관련 소식 상위 3 |
| 전체 피드 | `#recent` (+`#unread`,`#readAll`) | `/api/recent` |
| 근무기록 | `#wlSummary`,`#wlSub`,`#wlDays`,`#wlKm`,`#wlName`,`#wlCar`,`#wlSave`,`#wlExport`,`#wlReport` | `/api/worklog` |
| 하단 탭 | `nav.nav > button[data-view=today|news|worklog]` | 뷰 전환 |
| 뷰 | `#view-today`,`#view-news`,`#view-worklog` | 표시/숨김 |

→ 상세 데이터 형태는 [CONTRACTS.md](CONTRACTS.md).

## 미해결 디자인 과제 (디자인 AI에게)

- **스페어 대기 상태 화면**(매일 보는 상태인데 프로토타입이 안 그림). 시간 없이도 안심되게: 순번 큐 시각화 등.
- **다크모드**(새벽·야간용). 확정 프로토타입엔 없음 — 넣을지 결정 필요.
- **글자 크기 스케일**: 프로토타입이 296px 목업 기준이라 실제 폰에서 작을 수 있음.
- 행동 보드의 "출발 알림 예정" 문구는 Phase 2(실제 푸시) 전까진 자리표시.
