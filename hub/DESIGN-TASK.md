# 디자인 AI 인계서 — 부족한 화면 제작 요청

> **수신:** 디자인 담당 AI
> **발신:** 구현 담당(Claude Code)
> **의뢰인:** 김홍구 (리버힐 3부 캐디, 이 앱의 사용자 본인)
> **목적:** 확정 시안(행동 진행 보드)은 이미 앱에 이식됐다. 그런데 **매일 보는 상태가 미설계**이고 몇 가지 결정이 남았다. 이 부분들을 제작해 달라.

## 0. 30초 배경

- 앱: 골프장 캐디(김홍구)의 근무·티오프를 감시해 폰으로 알려주는 PWA. 폰 세로 전용, 한국어.
- 확정 UI = **행동 진행 보드**(`sandbox/hybrid-prototypes.html`의 HYBRID 02). 이미 구현·이식됨.
- **문제:** 그 프로토타입은 "근무 확정 + 티오프 있음" 한 상태만 예쁘게 그렸다. 하지만 김홍구님의 **일상 상태는 대부분 "스페어 대기"(티오프 없음)** 라서, 매일 보는 화면이 밋밋한 안내문으로 처리돼 있다. 이게 가장 큰 구멍이다.

## 1. 만들어야 할 것 (우선순위)

### 🔴 P0 — 스페어 대기 상태 화면 (가장 중요, 매일 보는 것)
- 상황: 근무 확정 전, **티오프 시각이 없음**. 아는 값은 **순번(예: 6번)**, "3부 스페어", (가끔) "최근 확정 커트라인".
- 요구: 시각을 **지어내지 말 것**(없는 티오프를 만들면 안 됨 — 과거 오발송 사고의 원인). 대신 **대기 자체를 안심되게** 시각화.
  - 예: 순번 큐/진행 느낌, "앞에 몇 명"(커트라인 발표됐을 때만), "확정되면 집에서 나갈 시각을 계산해 알려드림" 같은 기대 안내.
  - 행동 보드와 **같은 톤·크기감**을 유지하되 시간 대신 순번·대기를 주인공으로.

### 🟠 P1 — 모든 상태를 한 세트로
아래 **7개 상태**를 각각 완성해 달라(현재 각 상태의 데이터는 [CONTRACTS.md](CONTRACTS.md)의 status 표):
1. 근무 확정 + 티오프 있음 (프로토타입에 있음 — 이걸 기준 톤으로)
2. 근무 확정 + **티오프 아직 없음** ("근무 확정 · 티오프 배정 대기")
3. **스페어 대기** (P0)
4. 지금 차례(your_turn) — 최고 강조
5. 휴무(off) — 조용
6. 정보 없음(empty)
7. 로딩 / 오류 (오류를 휴무로 표현 금지)

### 🟠 P1 — 글자 크기 스케일 정상화
- 프로토타입은 폭 296px 목업 안에서 그려져 폰트가 9~11px로 작다. **실제 폰(360~430px) 기준**으로 위계를 다시 잡아 달라(핵심 숫자는 크게, 본문 가독).

### 🟡 P2 — 다크모드 (넣을지 포함해 제안)
- 캐디는 새벽·야간에 폰을 본다. 다크모드가 유용. 넣는다면 상태 색(근무 초록/스페어 주황/차례 빨강)이 두 테마에서 다 살아있게.

### 🟡 P2 — 온보딩 · 빈 상태 · 알림 켜기
- 첫 사용자가 "왜 알림을 켜야 하는지" 납득하도록. 빈 피드도 감시가 살아있다는 안심을 주게.

### ⚪ P3 — 피드 가독성 · 접근성 마감
- 소식 배지 정리, 시간 그룹핑(오늘/어제), 포커스 상태, 대비.

## 2. 산출물 형식 (이대로 주면 구현 AI가 CSS를 한 글자도 안 고치고 이식)

**두 개의 자체완결 HTML 파일**을 만들어 주세요:

1. **`index.html`** — 실제 배포되는 앱 화면(마크업 + `<style>`). [FRONTEND-CONTRACT.md](FRONTEND-CONTRACT.md)의 **요소 ID를 그대로 유지**(그래야 구현 AI가 데이터를 꽂음). 기본 레이아웃 1벌.
2. **`states.html`** — 위 **7개 상태를 각각 렌더한 레퍼런스**(같은 CSS 사용, 상태별로 섹션 나눠 실제처럼 채움). 구현 AI가 이걸 보고 `app.js`에서 상태별 화면을 만든다.

> 목업(그림)이 아니라 **진짜 파일**로. 스크린샷 말고 HTML/CSS 코드로 주세요.

## 3. 반드시 지킬 제약

- **자체완결**: 외부 CDN·웹폰트 URL 금지. 한글 = 시스템 폰트 스택(`-apple-system, "Malgun Gothic", ...`). 한글 웹폰트는 수 MB라 금지.
- **폰 세로 전용**, 본문 최대폭 ~600px, 360px에서 가로 스크롤 금지.
- **접근성**: 터치 ≥44px, 색만으로 상태 구분 금지, 시각 숫자 tabular-nums, `prefers-reduced-motion` 존중.
- **요소 ID 유지**([FRONTEND-CONTRACT.md](FRONTEND-CONTRACT.md) 표). 바꿔야 하면 그 표도 함께 갱신해 알려주세요.
- **기존 기능 보존**: 하단 내비(오늘/소식/근무기록), 알림 구독, 소식 피드, 근무기록(사진·CSV·PDF).
- **정확도 철학**: 불확실한 시간·상태를 지어내지 않는다 → 그런 자리는 "확인 필요/대기"로 정직하게.

완성한 `index.html` + `states.html`을 **김홍구님에게 전달** → 구현 AI가 그대로 이식하고, ID·데이터가 안 맞는 부분만 김홍구님과 확인 후 배선한다. (구현 AI는 CSS·구조를 임의로 바꾸지 않는다.)
