# 리버힐 캐디 근무 시스템 브리핑 — 스페어 대기 화면 설계용

> **수신:** 디자인 AI (스페어 대기 대시보드 제작 중)
> **발신:** 구현 담당(Claude Code) · **의뢰인/도메인 전문가:** 김홍구 (리버힐 3부 캐디 본인)
> **목적:** 스페어 대기 화면을 "겉모습"이 아니라 **실제 근무 현실**에 맞게 설계할 수 있도록, 이 근무 시스템이 어떻게 굴러가는지 브리핑한다.

## 1. 한 문장 요지 (설계의 핵심 감정)

> 캐디의 하루는 **"오늘 내가 나가나? 언제?"를 모른 채 기다리는 불확실성**이다. 스페어 대기 화면의 임무는 **그 불안을 줄이는 것** — 내가 지금 어디쯤 있고, 얼마나 근접했고, 언제 알 수 있는지를 한눈에.

김홍구님이 직접 한 말: *"매일 대기의 불확실성이 피로하다."* 이 화면은 그 피로를 덜어주는 도구다.

## 2. 부(部) 시스템

- 골프장 캐디는 **1부·2부·3부**로 나뉜다. **각 부는 완전히 독립** — 다른 부 소식은 나와 무관.
- **김홍구님 = 3부.**
- 부별 티오프(경기 시작) 시간대:
  - **1부 = 오전 이른 시간** (아웃/인 6~9시대)
  - **2부 = 낮** (대략 10~15시)
  - **3부 = 티오프 16시 이후** (저녁까지)
- 그래서 "아웃 7시33분" 같은 오전 소식은 1부, "인 13시35분"은 2부 → 3부인 김홍구님과 무관.

## 3. 캐디의 신분/상태 — 배치표 배경색이 진실

매일 **배치표**(캡처 이미지)가 올라오고, 각 캐디의 **이름칸 배경색**이 그날 신분을 나타낸다:

| 배경색 | 뜻 |
|---|---|
| ⬜ **흰색** | **근무 확정** (팀 매칭됨, 티오프 배정) |
| ⬛ **회색** | **스페어 = 대기** |
| 🟩 녹색 (54) | 1·2·3부 전부 근무 (18홀×3) |
| 🟦 하늘색 (2,3) | 2·3부 근무 |

- **핵심: 흰색이면 근무 확정.** 티오프 시각이 표에 아직 안 보여도, 흰색이면 근무 확정이다(임시 티오프가 곧 매칭됨).
- **회색 = 스페어.** 이 화면이 다루는 주 상태.

## 4. 순번(順番) 시스템 — 대기열

- 각 부에는 **순번 목록**이 있다. 스페어는 이 **순번 순서대로** 근무로 활성화된다.
- **순번은 고정이다.** 김홍구님이 직접 변경 요청을 하지 않는 한 안 바뀐다. (드물게 누군가 피치 못할 사정으로 순번이 앞당겨지는 경우는 있지만 거의 없음.)
- 순번은 배치표의 "OUT n부 IN" 티오프표(또는 순번 목록)에서 확인된다. 예: 김홍구님 = 3부 순번 **6번**.
- **순번 낮음(예 6번) = 근무 가능성 높음**, 순번 높음(예 30번) = 그날 팀이 많이 차야 도달. 단, **순번만으론 "얼마나 근접했는지"를 알 수 없다** — 그날 팀이 얼마나 차는지(커트라인)에 달림.

## 5. 스페어가 근무로 바뀌는 과정 (이 화면의 심장)

하루 동안 팀이 차거나, 당일 추가 팀이 생기거나, 취소·노쇼가 나면서 **대기열이 위에서부터 근무로 활성화**된다.

- **커트라인 소식:** "○○님까지 근무/나갑니다/일됩니다" = **○○(포함) 및 그 앞 순번 전원이 근무 확정**.
  - 내 순번이 ○○보다 **뒤**면 → 아직 대기.
  - 내 순번에 **도달/초과**되면 → **지금 출근!** (이때가 알림의 핵심 순간)
- **예시:** 커트라인 "28번 강성일님까지 근무" 이고 김홍구님이 **30번**이면 → 29, 30만 남음 → **앞으로 2명**, 거의 임박.
- **비순차·실시간·번복이 잦다:** 소식이 불특정하게 올라오고, 취소로 커트라인이 뒤로 물러나기도, 추가로 확 당겨지기도 한다. "확정됐다가 취소" 같은 번복도 있다. → 화면은 **최신 상태**를 보여주고, 바뀌면 강조.
- **순번 교환:** 가끔 두 사람이 순번을 맞바꾼다(당일 번호표에 '이름(다른이름)'으로 표시). 이때 내 실제 순번이 바뀔 수 있다.

## 6. 확정되면 무엇이 바뀌나 (스페어 → 근무)

스페어가 근무로 확정되면:
1. 신분이 **근무 확정**(흰색)이 되고
2. **티오프 시각**(HH:MM)과 **코스**(OUT/IN)가 배정된다
3. 거기서 역산: **골프장 도착 = 티오프 − 준비(약 1h)**, **집 출발 = 도착 − 이동(약 1h)**
   - 예: 티오프 17:21 → 16:21 도착 → 15:21 집 출발
4. → 이때부터는 **"행동 진행 보드"**(이미 만든 근무 확정 화면)로 넘어간다. 스페어 화면의 목표는 **여기까지 가는 대기 구간을 안심되게** 보여주는 것.

## 7. 언제 불확실성이 풀리나 (설계에 중요)

스페어는 **둘 중 하나**로만 결말을 안다:
- (A) **커트라인이 내 순번에 도달** → 근무 확정 (그날 안에)
- (B) **그날이 다 지나 다음날 배치표가 뜸** → 결국 근무 없었음(=그날 스페어로 끝)

즉 **대기 중엔 "확정도 탈락도 아닌" 회색지대**에 오래 머문다. 화면은 이 회색지대에서 **"지금 얼마나 근접했나 / 언제쯤 판가름 나나"**를 계속 알려줘야 초조함이 준다.

## 8. 소식이 오는 채널

- **네이버 카페:** 배치표(전날/당일), 당일 추가·변동·취소 글.
- **카카오톡 단톡방:** 실시간 커트라인 소식("○○님까지 나갑니다") 등. 카페보다 빠를 때가 많음.
- 앱이 둘 다 감시해서 대기열 상태를 갱신한다.

## 9. ★ 스페어 대기 화면에 담아야 할 정보 (디자이너 번역)

우선순위 순 — 대기 중인 캐디가 실제로 궁금한 것:

1. **내 순번** (예: 6번) — 고정된 나의 기준점. 늘 보이게.
2. **현재 커트라인** — "○○님까지 확정"(발표됐을 때만). 대기열이 어디까지 왔나.
3. **앞으로 몇 명** — 내 순번 − 커트라인 순번 (커트라인 발표됐을 때만). **없으면 지어내지 말 것.**
4. **근접도/긴장도** — 커트라인이 내 순번에 가까워질수록 시각적으로 고조(예: 진행 게이지, 색 변화). 도달 직전엔 "곧 차례" 느낌.
5. **마지막 갱신 시각 / 감시 살아있음** — "N분 전 확인" (안심 요소).
6. **기대 안내** — "확정되면 집에서 나갈 시각을 계산해 바로 알려드립니다" 같은, 다음에 뭐가 오는지.
7. (선택) **오늘 3부 진행 맥락** — 예상 팀 수 대비 지금 어디쯤(있으면).

## 10. ★ 절대 하지 말 것 (과거 실패에서)

- ❌ **스페어인데 티오프 시각을 지어내지 말 것.** (배정 전엔 시각이 없음. 지어내면 오알림 사고 — 실제로 "17시" 헛것을 보내 크게 혼났음.)
- ❌ **"쉬세요/휴무"로 표현 금지.** 스페어는 **대기**다(당일 근무로 바뀔 수 있음). 휴무(off)는 완전히 다른 상태.
- ❌ **순번 숫자만으로 "근접"을 단정하지 말 것.** 6번이라도 그날 팀이 적으면 대기, 30번이라도 팀 많으면 근무. 근접도는 **커트라인 대비**로만.
- ❌ 커트라인 발표가 없을 땐 "앞으로 N명"을 **계산하지 말 것**(모르면 "아직 확정 전"으로 정직하게).

## 11. 실제 예시 (2026-07-18 배치표 기준, 감을 잡으라고)

- 김홍구님: 3부 **순번 6번**, 이름칸 **흰색 → 근무 확정**, 티오프 **17:21 OUT** (그날은 확정 상태였음).
- 만약 아직 **회색(스페어)** 이고 커트라인이 "3번까지 근무"였다면 → 6번인 나는 **앞으로 3명**(4,5,6) 남은 대기.
- 커트라인이 "22번까지 근무"로 점프하면 → 6번은 이미 포함 → **근무 확정**으로 전환(번복 알림).
