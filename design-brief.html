리버힐 캐디 일정 앱 — 다음은 UI·경험 설계

기능은 안정화 단계에 들어섰다. 이 문서는 앱의 현재 모습·데이터·제약을 한눈에 펼쳐, 다른 AI나 디자이너가 곧바로 개선 설계에 참여할 수 있게 만든 공용 브리프다.

01

이 앱이 하는 일

골프장 캐디의 하루는 대기의 불확실성으로 채워진다. "오늘 내가 나가나? 몇 시에? 언제 집에서 출발하지?" — 이 질문에 스스로 답하려고 네이버 카페와 카톡 단톡방을 끝없이 새로고침한다. 앱의 단 하나의 임무는 그 불확실성을 대신 감시해, 내 차례·내 시간이 정해지는 순간 폰으로 먼저 알려주는 것이다.

감시원
네이버 카페 + 카톡3부 배치표·당일 추가·커트라인 소식을 자동 수집
두뇌
이미지·텍스트 판독배치표 사진에서 내 순번·티오프를 읽고, "○○님까지" 소식으로 내 차례 계산
알림
웹푸시(FCM)근무 확정·차례 임박·변경(번복)을 폰으로 즉시
덤
근무·세무 기록근무 확정일 자동 기록 → 차량 운행일지·계기판 사진 증빙 문서

03

① 오늘 내 상황 (상황판 히어로)

04

지금 부족한 것 — UX 진단

우선순위 순. P0 신뢰가 최우선인 이유는, 방금 죽은 구독으로 알림이 몇 건이나 조용히 새어나간 사고를 겪었기 때문이다. 알림 앱은 "안 오는 것"이 최악의 실패다.

P0 · 신뢰

알림이 실제 도착하는지 앱이 안 보여준다

  • 감시(서버가 카페를 보는지)는 표시하지만, 전달(이 폰이 푸시를 받을 수 있는지)은 어디에도 없다.
  • 구독이 만료돼 끊겨도 사용자는 알 방법이 없다 — 그냥 조용해질 뿐. 실제로 그렇게 여러 건을 놓쳤다.
  • 필요: "이 기기 알림 정상/끊김" 지표 + 끊기면 원터치 재등록 유도 + (선택) 마지막 도착 확인 시각.
P1 · 한눈

상황판이 히어로인데 텍스트로만 말한다

  • 가장 중요한 질문 "언제 집에서 나가야 하나"가 한 줄 텍스트에 묻힌다.
  • 없는 것: 카운트다운("집 출발까지 2시간 15분"), 타임라인 스트립(지금 → 집출발 → 도착 → 티오프), 스페어 큐 시각화("30번 중 앞에 N명").
  • 대기의 불확실성을 없애는 게 앱의 존재 이유 → 이 카드가 감정적으로도 "안심"을 줘야 한다.
P1 · 구조

기능이 늘었는데 내비게이션이 없다

  • 상황판·피드·근무기록·세무증빙이 한 스크롤에 전부 섞여 길다.
  • 하단 탭(예: 홈 · 소식 · 기록) 또는 명확한 구획으로 분리 필요.
  • 매일 보는 건 상황판·피드, 가끔 쓰는 건 세무 — 사용 빈도로 위계를 나눠야 한다.
P2 · 첫인상

온보딩·빈 상태가 빈약하다

  • 첫 사용자는 맥락 없이 "알림 켜기" 버튼만 본다 — 왜 켜야 하는지 설득이 없다.
  • 빈 피드는 "아직 감지된 소식이 없어요" 한 줄. 감시가 살아있다는 안심을 함께 줘야.
P2 · 환경

다크모드가 없다

  • 캐디는 이른 새벽·늦은 저녁에 폰을 본다(3부 티오프는 16시 이후). 밝은 화면만 있다.
  • 야외·역광 가독성, 눈부심 모두 다크모드가 해결한다.
P3 · 밀도

근무·세무 섹션이 강력하지만 빽빽하다

  • 작은 입력칸, 사진 3슬롯, 계기판 숫자칸이 인라인으로 촘촘 — 별도 화면으로 숨 쉴 공간 필요.
  • 피드 배지 종류가 많고 규칙이 눈에 안 들어온다. 시간 그룹핑(오늘/어제)도 없다.
P3 · 마감

접근성·피드백 디테일

  • 키보드 포커스 상태 없음, 일부 muted 텍스트 대비 약함, 로딩 스켈레톤 없음, 저장 성공 피드백이 약하다.
05

참여하는 이에게 — 디자인 과제

개선 제안은 이 질문들에 답하는 형태면 좋다. 정답은 하나가 아니다.

Q1

상황판 히어로 재설계

대기의 불확실성을 없애는 "한눈" 화면. 카운트다운·타임라인·큐 위치를 어떻게 담을까? 4가지 상태 모두 우아하게.

Q2

정보 구조

한 스크롤을 유지할까, 하단 탭으로 나눌까? 매일/가끔 쓰는 기능의 위계는?

Q3

신뢰 레이어

감시(서버)와 전달(구독) 건강을, 불안하지 않고 안심되게 보여주는 방법. 끊겼을 때의 복구 흐름.

Q4

다크모드 + 야외 가독성

새벽·야간·역광에서 읽히는 이중 테마. 상태 색(초록/주황/빨강)이 두 테마에서 다 살아있게.

Q5

세무 기록 간소화

사진 3장·계기판 입력을 부담 없이 만드는 흐름. 별도 화면? 촬영 유도?

Q6

정체성

지금은 이모지 위주. 골프장 캐디의 세계에 맞는 고유한 톤·타이포·컬러가 있다면?

06

지켜야 할 제약

디자인은 이 경계 안에서 자유롭게. 경계를 넘는 제안(예: 네이티브 앱, 로그인 시스템)은 별도 논의가 필요하다.

PWA · 안드로이드 크롬홈 화면 설치형 웹앱. iOS는 후순위.
폰 세로 전용본문 최대폭 640px. 데스크톱 고려 불필요.
한국어 · 시스템 폰트한글 웹폰트는 수 MB라 비현실적. 위계는 굵기·크기로.
단일 사용자김홍구 1인용. 로그인·다중계정 없음.
정적 자체완결public/의 index.html·app.js·sw.js. 외부 CDN 불가.
실시간 아님30초 폴링으로 상황판·피드 갱신. 웹푸시는 즉시.

07

현재 디자인 시스템 — 존중하되 진화시켜도 좋다

기존 앱은 페어웨이 그린을 브랜드색으로 쓴다. 상태 색(근무=초록, 스페어=주황, 차례=빨강, 휴무=회색)은 의미가 굳어 있으니 의미는 유지하되 색조·명도는 다듬어도 된다.
