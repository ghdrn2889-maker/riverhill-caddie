# 리버힐 캐디 일정 앱 — UI·UX 샘플 제작 방향

> 상태: 디자인 제작 전 임시 제안서  
> 목적: 실제 앱에 적용하기 전에 성격이 다른 3개의 모바일 UI 샘플을 비교한다.  
> 작업 범위: 이 문서와 이후 샘플은 `sandbox` 안에서만 제작하며, 현재 `public` 앱은 수정하지 않는다.

## 1. 이번 디자인에서 해결할 문제

이 앱은 일반적인 일정 관리 앱이 아니다. 사용자가 가장 빠르게 알고 싶은 것은 “오늘 내가 출근하는가”, “몇 시까지 어디로 가야 하는가”, “알림은 지금도 정상적으로 도착하는가”이다. 따라서 기능의 개수보다 판단 속도와 신뢰가 중요하다.

현재 화면의 정보는 충분하지만 다음 문제가 있다.

- 오늘의 상태가 텍스트 중심이라 급히 볼 때 핵심 판단이 늦다.
- 서버 감시 상태와 휴대폰 알림 수신 상태가 구분되지 않는다.
- 일정 소식, 알림 설정, 근무·세무 기록이 한 화면에 이어져 사용 빈도에 따른 위계가 약하다.
- 야외 역광과 새벽·야간 환경을 고려한 대비와 터치 영역이 부족하다.
- 이모지에 의존해 앱만의 시각적 정체성이 약하다.

## 2. 세 샘플에 공통으로 적용할 원칙

### 2.1 첫 화면의 우선순위

첫 화면은 아래 순서를 유지한다.

1. 오늘의 상태: 근무 확정, 턴 대기, 스페어 대기, 휴무
2. 다음 행동: 출발 시각, 티오프, 대기 순번 또는 확인할 내용
3. 알림 신뢰 상태: 정상, 확인 필요, 끊김
4. 최근 일정 소식
5. 근무·세무 기록

### 2.2 상태를 색상에만 의존하지 않기

초록·주황·빨강은 의미를 빠르게 전달하지만 아이콘, 제목, 짧은 설명을 함께 사용한다. 색각 차이가 있거나 햇빛 아래에서 색 구분이 약해져도 상태를 이해할 수 있어야 한다.

### 2.3 알림 상태를 두 층으로 표시하기

- 일정 감시: 서버가 게시판을 정상적으로 확인하고 있는가
- 알림 전달: 현재 휴대폰이 푸시 알림을 실제로 받을 수 있는가

정상일 때는 조용하고 간결하게 보여주며, 문제가 있을 때만 원인과 복구 버튼을 강조한다.

### 2.4 모바일 현장 사용성

- 주요 텍스트는 한 손 거리에서도 읽히는 크기로 만든다.
- 핵심 버튼의 최소 높이는 48px로 한다.
- 숫자와 시간은 자릿수가 흔들리지 않는 서체 표현을 사용한다.
- 야외용 고대비 라이트 테마와 눈부심을 줄인 다크 테마를 함께 설계한다.
- 중요한 동작에는 즉시 완료·실패 피드백을 제공한다.

## 3. 샘플 A — “클럽하우스 보드”

### 한 줄 설명

골프장 출근 직전에 보는 전광판처럼, 오늘 필요한 정보만 크고 선명하게 보여주는 실용 중심 디자인.

### 제작 의도

사용자가 앱을 열자마자 3초 안에 오늘의 상태와 다음 행동을 판단하도록 한다. 장식보다 정보 대비를 우선하며 현재 앱과 가장 자연스럽게 연결되는 안정적인 개선안이다.

### 시각 방향

- 페어웨이 딥그린, 아이보리, 차콜을 기본 색으로 사용
- 큰 상태 제목과 굵은 시간 숫자
- 카드 수를 줄이고 섹션 경계를 선과 여백으로 명확히 표현
- 골프장 스코어보드에서 가져온 절제된 라벨과 숫자 표현
- 이모지 대신 단순한 선형 아이콘 사용

### 화면 구조

- 상단: 날짜, 날씨 자리, 앱 설정
- 히어로: 오늘 상태, 티오프/출발 시간, 장소, 남은 시간
- 신뢰 표시줄: “일정 감시 정상 · 이 휴대폰 알림 정상”
- 최근 소식: 중요도와 새 소식 여부가 분명한 목록
- 하단 내비게이션: `오늘 / 소식 / 기록`

### 장점

- 빠르게 이해할 수 있고 야외 가독성이 좋다.
- 기존 브랜드 색과 기능을 가장 적은 학습 비용으로 발전시킨다.
- 실제 구현 난도가 낮고 PWA 성능에 부담이 적다.

### 주의점

- 지나치게 실용적으로 만들면 감성적 인상이 약할 수 있다.
- 상태 정보가 적은 날에는 화면이 다소 비어 보일 수 있다.

## 4. 샘플 B — “캐디 데이 타임라인”

### 한 줄 설명

오늘 하루의 흐름을 출발–도착–티오프–종료 순서로 보여주는 행동 안내형 디자인.

### 제작 의도

단순히 일정이 있다는 사실보다 “지금 무엇을 준비해야 하는지”를 안내한다. 특히 근무가 확정된 날과 턴 대기 중인 날의 불확실성을 줄이는 것이 목표다.

### 시각 방향

- 밝은 세이지 그린과 따뜻한 샌드 컬러
- 세로 타임라인, 진행 점, 남은 시간 링을 핵심 그래픽으로 사용
- 카드 모서리는 부드럽게, 그림자는 최소화
- 안내 문장은 짧고 대화하듯 표현
- 상태 변화에는 과하지 않은 전환 애니메이션 사용

### 화면 구조

- 상단: 인사말과 오늘 날짜
- 히어로: “출발까지 1시간 20분”처럼 행동 중심 메시지
- 오늘 타임라인: 집 출발, 골프장 도착, 티오프, 종료 예상
- 대기 상태일 때: 현재 순번과 다음 확인 예상 시점
- 최근 변화: 일정 변경과 새 공지를 시간순으로 표시
- 기록 유도: 근무 종료 후 사진 및 계기판 기록을 단계별로 안내

### 장점

- 사용자가 다음 행동을 놓칠 가능성이 가장 낮다.
- 사진 3장과 운행 기록 흐름을 자연스럽게 연결할 수 있다.
- 앱이 단순 조회 도구가 아니라 현장 보조 도구처럼 느껴진다.

### 주의점

- 정확한 출발·도착 계산 데이터가 부족하면 일부 정보를 추정해야 한다.
- 휴무나 소식이 없는 날에도 자연스러운 빈 상태를 별도로 설계해야 한다.

## 5. 샘플 C — “포켓 컨시어지”

### 한 줄 설명

개인 캐디 매니저처럼 핵심 상황을 요약하고 필요한 순간에만 기능을 꺼내 보여주는 프리미엄 미니멀 디자인.

### 제작 의도

단일 사용자를 위한 앱이라는 특성을 적극 활용한다. 복잡한 메뉴보다 개인화된 한 문장과 상황별 빠른 행동을 앞세워, 매일 부담 없이 확인하고 오래 사용할 수 있는 인상을 만든다.

### 시각 방향

- 다크 포레스트, 크림, 브라스 골드를 사용한 차분한 팔레트
- 넓은 여백과 큰 문장형 헤드라인
- 숫자와 시간에 에디토리얼 스타일의 강한 위계 적용
- 아이콘은 최소화하고 작은 상태 점과 정제된 선을 사용
- 라이트·다크 테마 모두 고급스럽고 차분하게 구성

### 화면 구조

- 상단: “홍구님, 오늘은 3부 스페어 대기입니다” 요약
- 중심 카드: 가장 중요한 한 가지 행동과 시간
- 상태 캡슐: 감시 및 알림 정상 여부
- 펼침 영역: 최근 소식과 상세 근거
- 별도 기록 화면: 월간 근무일, 주행 거리, 증빙 완성도
- 설정은 우측 상단의 작은 메뉴로 이동

### 장점

- 앱만의 정체성과 완성도가 가장 강하다.
- 정보가 많지 않은 날에도 화면이 안정적으로 보인다.
- 단일 사용자에게 맞춘 친밀한 문구를 적극 활용할 수 있다.

### 주의점

- 미니멀한 표현이 익숙하지 않으면 일부 기능을 찾기 어렵게 느낄 수 있다.
- 골드 계열 강조색은 상태 경고색과 혼동되지 않도록 세심한 대비 조정이 필요하다.

## 6. 세 방향 비교

| 비교 항목 | A. 클럽하우스 보드 | B. 데이 타임라인 | C. 포켓 컨시어지 |
|---|---|---|---|
| 핵심 가치 | 즉시성·가독성 | 행동 안내·과정 | 개인화·정체성 |
| 정보 밀도 | 중간~높음 | 중간 | 낮음~중간 |
| 기존 앱과의 연속성 | 매우 높음 | 높음 | 중간 |
| 야외 가독성 | 매우 높음 | 높음 | 높음 |
| 세무 기록 확장성 | 보통 | 매우 높음 | 높음 |
| 디자인 개성 | 보통 | 높음 | 매우 높음 |
| 예상 구현 난도 | 낮음 | 중간~높음 | 중간 |

## 7. 샘플 제작 범위

각 방향은 모바일 화면을 기준으로 동일한 내용을 담아 비교한다.

- 오늘이 `3부 스페어 대기`인 기본 홈 화면
- `근무 확정`으로 상태가 바뀐 홈 화면
- 알림 구독이 끊겼을 때의 경고 및 복구 상태
- 최근 소식 3건
- 근무·세무 기록 진입 방식
- 라이트 테마 기본안과 다크 테마 핵심 색상

샘플은 실제 API와 연결하지 않는 정적 프로토타입으로 만든다. 버튼과 탭에는 화면 전환 정도의 상호작용을 넣어 구조와 사용감을 확인할 수 있게 한다.

## 8. 추천 진행안

첫 비교에서는 A를 구현 기준선으로 삼고, B의 타임라인/촬영 흐름과 C의 개인화 문구/시각 정체성을 함께 검토하는 방식을 추천한다. 최종안은 세 샘플 중 하나를 그대로 선택하기보다 실제 사용에 맞는 강점을 조합할 수 있다.

평가할 때는 “가장 예쁜 화면”보다 아래 질문을 우선한다.

1. 앱을 켠 즉시 오늘 할 일을 이해할 수 있는가?
2. 알림이 정상이라는 확신을 얻을 수 있는가?
3. 야외에서 한 손으로 필요한 동작을 끝낼 수 있는가?
4. 근무 후 기록이 귀찮지 않게 이어지는가?
5. 매일 보아도 피로하지 않은가?
